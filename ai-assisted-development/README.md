# AI-Assisted Development

This folder documents how I am integrating AI into my day-to-day learning and development process in a practical, responsible way.

## Why this exists

I am currently learning Azure and Cloud through a structured mentoring process. At the same time, I want to build strong AI-assisted work habits so I can stay current in the market without compromising technical depth.

My goal is **not** to outsource thinking to AI. My goal is to use AI as a copilot that helps me:

- learn faster,
- practice more consistently,
- improve communication and documentation quality,
- and keep my focus on understanding systems, not just producing outputs.

## What this section contains

- `tools-and-workflows.md`: a living document of:
  - tools I currently use,
  - tools I am evaluating,
  - usage principles,
  - learning workflows supported by AI,
  - and guardrails for responsible use.

## How to read this folder

I keep this content intentionally honest and experience-based:

- If I already use a tool, I document real usage.
- If I have not used a tool yet, I list it as a candidate and explain why it might be useful.
- I update this section as my workflow evolves.
